The hydration error is occurring because the theme script in your `layout.tsx` is running on the client and potentially modifying the DOM before React hydrates, causing a mismatch between server-rendered HTML and client-rendered HTML.

The issue is that the inline script sets the `data-theme` attribute on `document.documentElement` based on `localStorage`, which only exists in the browser. This means:
- Server renders with no `data-theme` attribute (or a default one)
- Client script runs and sets `data-theme` from localStorage
- React tries to hydrate and finds a mismatch
